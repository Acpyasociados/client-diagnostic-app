## Reglas para Testing

### Campos Requeridos
```
✓ Todos deben estar llenos
✓ Email debe tener @
✓ Números deben ser positivos
✓ Sector debe ser seleccionado
✓ Desafío debe ser seleccionado
✓ Asesor debe ser seleccionado
✓ Plan debe ser seleccionado
```

### Campos Condicionales
```
✓ Si Sector = "Otro" → sector_other requerido
✓ Si Desafío = "Otro" → main_challenge_other requerido
✓ Si sector tiene mapeo → campos operacionales requeridos
```

### Validaciones
```
✓ Email: formato válido (contiene @)
✓ Números: mayores a 0
✓ Teléfono: cualquier formato
✓ Texto: no vacío
✓ Margen: 0-100 (HTML5, aunque backend puede validar)
```

## Notas de Testing

1. **Sector "Otro":** Cuando selecciones "Otro", aparece campo texto → llena con valor personalizado
2. **Campos Dinámicos:** Cambia sector varias veces → los campos deben actualizarse
3. **Desafío "Otro":** Igual que sector → campo condicional
4. **MRR Opcional:** Si sector es Tecnología, MRR puede estar vacío
5. **JSON:** Campos operacionales se envían como JSON string en form-urlencoded

---

## Errores Esperados (Negativo Testing)

```
Envío sin nombre:
→ Error: "Por favor ingresa tu nombre"

Envío con email inválido (sin @):
→ Error: "Por favor ingresa un email válido"

Sector "Otro" sin especificar:
→ Error: "Por favor especifica tu rubro personalizado"

Desafío "Otro" sin especificar:
→ Error: "Por favor especifica tu desafío personalizado"

Números operacionales vacíos (required):
→ Error: "Por favor completa [campo]"

Campo numérico con letras:
→ Error: "Por favor ingresa un número"
```
