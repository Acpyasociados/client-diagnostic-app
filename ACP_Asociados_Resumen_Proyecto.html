¿Qué es el Proyecto?
Propuesta de valor, planes y modelo de negocio

ACP & Asociados es una plataforma web que entrega diagnósticos empresariales personalizados con IA para PyMEs chilenas. El cliente contrata online, responde un cuestionario, y en 48–72 horas recibe un informe PDF con oportunidades de mejora específicas para su sector y tamaño de empresa.

💡 Diferenciador clave: No es un reporte genérico. El motor de IA (Claude Haiku) cruza 10 dimensiones de análisis con las respuestas del cuestionario para generar recomendaciones específicas al rubro y al tamaño de la empresa.

Arquitectura Técnica
Stack, funciones serverless y variables de entorno
Netlify (hosting + functions)
Serverless ES Modules. Scheduled functions para cron jobs.
Netlify Blobs
Base de datos de leads. Store: diagnostic-leads.
Flow
Pasarela de pagos chilena. Webhook HMAC-SHA256 verificado.
Resend API
Emails transaccionales. Key en SENDGRID_API_KEY.
Claude Haiku API
Enriquecimiento IA de informes. Key en ANTHROPIC_API_KEY.
Puppeteer + Chromium
Generación de PDF (Sparticuz/chromium).
GitHub API
Backup diario de leads a las 03:00 AM Chile.
HTML/CSS/JS vanilla
Frontend sin framework. admin.html para gestión interna.

Seguridad Implementada
Vulnerabilidades corregidas y protecciones activas
✅ Auditoría completa realizada. 5 vulnerabilidades críticas y 5 importantes fueron identificadas y corregidas. El sistema tiene defensa en profundidad.

Críticas — Corregidas

#VulnerabilidadCorrección
C-1Panel admin sin autenticaciónview-leads.js valida ADMIN_REVIEW_TOKEN antes de cualquier operación
C-2Precio aceptado desde el body del clientePrecio calculado 100% server-side con validateCouponServerSide()
C-3Cupones de prueba activos en producción (TEST100, DEMO100…)Eliminados. Solo quedan DESC50 y STARTUP30
C-4Admin token expuesto en URLs de emailEmails usan solo la URL del panel, sin token en query string
C-5Token admin en localStorage (persistente)Migrado a sessionStorage (se borra al cerrar el tab)

Importantes — Corregidas

#VulnerabilidadCorrección
I-1Webhook de pago sin verificación de montoCompara amount recibido vs lead.final_price; si hay diferencia marca fraud_detected
I-2Cuestionario sin verificar pagoBloquea si payment_status !== 'approved' — error 402
I-3Generar informe sin autenticaciónRequiere client_token (cliente) o admin_token
I-6Estado marcado como enviado antes de verificar emailEmail se envía primero; si falla se retorna error 500 sin cambiar estado
I-8Idempotencia del webhook basada en estado pipelineAhora verifica payment_status === 'approved' — más robusto

Guía de Replicación por País
Cómo lanzar el mismo modelo en otro mercado

El proyecto está diseñado para ser replicable. El único componente local es la pasarela de pagos. Todo lo demás (IA, emails, hosting, base de datos) es global.

Checklist de adaptación (≈12 horas de trabajo)

  • Crear cuenta en pasarela de pago local y obtener API keys
  • Reemplazar flow-webhook.js con el equivalente para la nueva pasarela
  • Actualizar create-diagnostic-order.js para apuntar al nuevo checkout
  • Actualizar precios en netlify.toml (PRICE_BASIC_CLP, PRICE_PREMIUM_CLP)
  • Ajustar textos de la landing page (país, moneda, IVA local)
  • Actualizar sectores si el mercado local tiene categorías distintas
  • Crear nuevo sitio Netlify y configurar las 8 variables de entorno
  • Registrar dominio local y configurar DNS en Netlify
  • Configurar email de envío en Resend (dominio verificado)
  • Test de punta a punta: comprar → cuestionario → informe → entrega
📌 El código de los motores de reglas (report.js) es reutilizable. Solo los precios, moneda y textos de la landing requieren adaptación.
